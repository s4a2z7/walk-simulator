# Step 1 산출물 — 데이터 수집 계획표

## 0. 목적/원칙
- 목적: 데모/평가에 필요한 데이터(업체/슬롯/기록 등)를 **일정 내 확보**하고, 통합 시 **스키마/정합성** 문제를 줄인다.
- 원칙:
  - 외부 제휴/계약이 필요한 소스는 Step 1에서 “의존”하지 않는다.
  - 핵심 기능(예약/플로우/상태 전이)은 **내부 Seed/모킹 데이터로도 검증** 가능해야 한다.
  - 개인 민감정보는 최소 수집(연락처 등) + 마스킹/가명 처리.

---

## 1. 데이터 도메인별 출처 조사 및 확보 전략(요약)

### 1.1 권장 소스 조합(제출/데모 기준)
- **기본**: 내부 Seed + 모킹(테스트 데이터셋 직접 생성)
- **선택**: 공공 데이터 또는 지도/장소 API로 업체 목록 현실감 보강

## 3. 데이터 수집/정제 상세(권장 작업)

### 3.1 내부 Seed 데이터(업체) 최소 기준
- 최소 10개(병원 4, 약국 3, 검진센터 3) + 지역 2개 이상
- 필수 필드: `provider_id`, `provider_type`, `name`, `address`, `phone(선택 가능)`, `services`
- 표기 규칙: 업체명/지점명 분리(`name`, `branch_name`)

### 3.2 슬롯(가용 시간) 생성 기준(데모용)
- 업체별 2~3일치, 하루 4~6개 슬롯
- 서비스 유형별(진료/검진/수령)로 최소 1개 이상 포함
- `remaining=0` 슬롯을 섞어 “마감(예외)” 시나리오 가능하게 구성
