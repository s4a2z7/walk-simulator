# 헬스케어 통합 챗봇 서비스 기획서(요약)

## 1. 서비스 개요 (3-5문장)
본 서비스는 사용자의 건강 관련 요청을 **대화형 인터페이스**로 받아, 기록·관리·실행(예약 등)까지 이어지도록 지원하는 **통합 헬스케어 챗봇**이다.  
기능은 팀 단위 모듈로 분리되어 있으며(예: 영양·알러지 관리, 활동/걸음 수, 의료업체 예약), 통합 단계에서 단일 챗봇 경험으로 제공한다.  
사용자는 영양/알러지 및 생활 데이터를 간단히 남기고, 필요 시 의료업체(병원/약국/검진센터) 방문을 예약해 후속 행동까지 연결할 수 있다.  
단, 서비스는 **의료 진단·처방을 수행하지 않으며**, 예약은 “업무 처리” 범위에서만 제공한다.

## 2. 핵심 가치 제안 (Value Proposition)
| 대상 | 고객 문제(Problem) | 제공 가치(Value) | 근거/특징(How) |
|---|---|---|---|
| 건강 관리 초보 사용자 | 무엇을 어떻게 기록하고, 언제 행동해야 하는지 흐름이 끊김 | **기록 → 다음 행동**으로 자연스럽게 이어지는 단일 플로우 제공 | 모듈 기능을 한 대화에서 연결(기록/조회/예약) |
| 바쁜 직장인/보호자 | 어느 기관이 맞는지/어디로 연락해야 하는지 탐색 비용이 큼 | **업체·방문유형·준비사항·연락 경로를 추천**해 행동 전환 비용을 감소 | 예약 추천 모듈이 조건 기반 추천 + 링크/전화 핸드오프 제공 |
| 팀 프로젝트/통합 관점 | 기능이 분산 구현되어 통합 시 충돌 가능 | **역할·책임 경계가 명확한 모듈 구조**로 통합 리스크 감소 | 단일 책임 원칙 기반(영양·알러지/활동/예약 분리) |

## 3. 주요 기능 목록 (5-7개)
1) **영양/알러지 기록·조회(타 모듈)**  
   - 식단/섭취 기록을 남기고, 알러젠/주의사항 프로필을 관리·조회한다.
2) **활동(걸음 수) 기록/조회(타 모듈)**  
   - 걸음 수 기반의 활동 데이터를 확인하고 추이를 본다.
3) **의료업체 통합 예약 추천(내 모듈)**  
   - 병원/약국/검진센터에 대해 방문 목적/조건을 받아 **적합한 기관과 이용 경로(전화/링크)**를 추천한다.
4) **증상 기반 방문 유형 추천(예약 보조, 제한적 AI)**  
   - 사용자의 증상을 근거로 “방문 유형/업체 카테고리” 선택을 돕되, 진단·처방은 하지 않는다.
5) **예약/기록 연계 컨텍스트 전달(통합 기능)**  
   - 다른 모듈에서 생성된 요약(예: 알러지 주의사항)을 예약 모듈로 전달해 입력 부담을 줄인다.
6) **알림/리마인드(통합 기능, 선택)**  
   - (예약 확정이 아닌) 추천 결과 저장/재확인, 기록 리마인드 등 사용자 행동을 유도하는 메시지를 제공한다.

## 4. 사용자 시나리오 (User Journey)
### 4.1 페르소나
- **직장인 A(29세)**: 평일에 병원 방문이 어렵고, 알러지/식단 이슈가 있을 때 필요한 예약 업무를 빠르게 처리하고 싶다.

### 4.2 여정 단계
| 단계 | 사용자 행동 | 시스템(챗봇) 동작 | 결과 |
|---|---|---|---|
| 1. 유입/시작 | “계란 알러지가 있어요. 프로필에 저장해줘” 입력 | 영양/알러지 관리 흐름으로 연결(타 모듈) | 알러지 프로필 저장 |
| 2. 다음 행동 연결 | “알러지 상담 때문에 병원 예약 추천받고 싶어요” 입력 | 예약 모듈로 전환 + 필요 시 **방문 유형 추천(예약 보조)** 제안 | 예약 전환 준비 |
| 3. 추천 받기 | “오늘 저녁 방문 가능한 곳 추천해줘” | 추천 조건 수집 → 후보 기관 3개 + 연락/경로 제공 | 추천 결과 확보 |
| 4. 조건 변경 | “좀 더 가까운 곳으로 바꿔줘” | 조건 수정 → 추천 재생성 | 추천 갱신 |
| 5. 사후 처리 | 방문 후 “다녀왔어요” 또는 시간 경과 | 완료 처리(규칙/확인 기반) | 예약 완료 상태 반영 |

## 5. 성공 지표 (KPI 5개)
1) **추천 수락률**: 추천 결과 제시 대비 사용자가 1개 이상 선택한 비율  
2) **핸드오프 전환율**: 추천 선택 대비 전화/링크 클릭(또는 복사) 비율  
3) **추천 처리시간**: 추천 요청 시작부터 결과 제시까지 중앙값(p50/p90)  
4) **추천 재요청률**: 최초 추천 후 조건 변경/재추천 비율(품질/커버리지 지표)  
5) **출처 커버리지**: 추천에 활용된 기관 메타/FAQ 텍스트의 소스 비중(공공/스크래핑) 및 최신성
