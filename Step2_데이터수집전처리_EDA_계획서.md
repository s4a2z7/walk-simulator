# Step 2 산출물 — 데이터 수집 및 전처리 + EDA 계획서(토~금)

## 0. Step 2 목표/산출물(요구사항 매핑)
| 트랙 | 목표 | 산출물(제출/세션 기준) |
|---|---|---|
| 데이터 수집 시작하기 | 도메인별 리뷰/문의/텍스트 데이터 확보 + 초기 품질 확인 | **Raw 데이터 샘플 100~500건** |
| 데이터 정제 및 전처리 | 결측치 제거/필터링 + 텍스트 정규화 + 토큰화/형식 통일 | **전처리 완료 데이터셋**, **전처리 코드** |
| 탐색적 데이터 분석(EDA) | 길이 분포/키워드 빈도/유형별 분포 + 시각화 | **EDA 리포트(시각화 포함)** |

---

## 1. 데이터 범위(도메인 정의)
### 1.1 텍스트 데이터 도메인(예시)
- **리뷰**: 방문 경험(대기/친절/설명/가격/청결 등)
- **문의(Q&A)**: 예약/변경/취소/준비물/운영시간/위치 등
- **업무 로그(선택)**: 데모 챗봇 대화 로그(PII 마스킹 전제)

### 1.2 최소 스키마(공통)
- `text_id`, `source_type(review|inquiry)`, `provider_type(HOSPITAL|PHARMACY|CHECKUP_CENTER)`
- `provider_name`, `created_at`, `text`
- 선택: `rating`, `sentiment_label`, `topic_label`

---

## 2. 일정(토~금) — 수행 계획(권장)
> 실제 날짜는 팀 캘린더에 맞춰 기입한다.

| 요일 | 목표 | 작업 | 산출물 |
|---|---|---|---|
| 토 | Step 2 킥오프 | 데이터 소스 후보 확정(리뷰/문의/로그), 스키마 확정 | 스키마 v1 |
| 일 | 수집 시작 | Raw 샘플 수집/생성(최소 100건) + 결측/중복 1차 확인 | Raw 100+ |
| 월 | 수집 확대 | 200~500건까지 확장 또는 도메인별 균형 조정 | Raw 200~500 |
| 화 | 전처리 구현 | 결측 제거/필터링 + 정규화 + 토큰화/형식 통일 | 전처리 코드 v1 |
| 수 | 전처리 검증 | 전처리 전/후 품질 지표 비교(결측률, 중복률, 길이 분포) | 전처리 데이터셋 v1 |
| 목 | EDA | 길이 분포/키워드 빈도/라벨 분포 시각화 | EDA 그림/표 |
| 금 | 리포트 정리 | EDA 결과 요약 + 다음 액션(라벨링/추가 수집) | EDA 리포트 |

---

## 3. 품질 체크 항목(최소)
### 3.1 Raw 데이터 체크
- [ ] 행 수: 100~500건 충족
- [ ] 결측치: `text`, `provider_type` 결측률 확인
- [ ] 중복: 동일 `text` 또는 동일 키(`provider_name+text`) 중복률 확인
- [ ] 길이: 너무 짧은 텍스트(예: 5자 미만) 비율 확인

### 3.2 전처리 후 체크
- [ ] 전처리 규칙이 재현 가능(코드로 고정)
- [ ] 토큰/정규화로 의미 손실이 과도하지 않음(샘플 20건 육안 확인)
- [ ] 저장 포맷 통일(CSV/JSONL 중 1개 고정)

## 4. 파일/코드 구성(이번에 제공되는 것)
- Raw 샘플: `data/step2_raw_text_sample.csv` (100+건)

- EDA 코드: `scripts/step2_eda_text.py`
- (선택) Raw 샘플 생성기: `scripts/generate_raw_text_sample.py`
- 출력 폴더: `out/` (전처리 결과/EDA 이미지 저장)
