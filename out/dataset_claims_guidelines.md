# 데이터셋 주장 가능 범위 가이드라인

## 1. 이 데이터셋으로 합리적으로 주장 가능한 것
- 중복 제거 및 기본 전처리가 완료되어 데이터 일관성이 확보되었음.
- EDA를 통해 텍스트 길이, 키워드 빈도, 감정 및 토픽 분포를 분석하여 데이터 특성을 이해할 수 있음.
- 의료 대화 중심으로 한국어 대화 패턴을 반영하며, LLM 학습용으로 적합한 구조적 텍스트를 제공함.
- 일반 대화 일부 포함으로 도메인 다양성을 일부 고려하였음.
- 품질 체크(중복, PII 등)를 통해 신뢰할 수 있는 데이터셋임을 입증함.
- 검색용으로 활용 시 의료 관련 쿼리에 대한 유사도 기반 검색이 가능함.

## 2. 현 단계에서 주장하면 위험한 것
- 이 데이터셋이 모든 한국어 대화 시나리오를 대표한다고 주장할 경우, 도메인 편중으로 인한 일반화 한계를 간과함.
- LLM 성능 향상을 보장한다고 주장할 경우, 323건 규모의 제한으로 충분한 학습 효과를 입증하기 어려움.
- 완벽한 데이터 품질을 주장할 경우, 의미적 유사나 라벨링 오류 등의 잠재적 리스크를 무시함.
- 확장 예정 데이터를 현재 성능으로 포함시키는 주장, 실제 수집 전 불확실성 존재.
- 검색 정확도를 100%로 주장할 경우, EDA 기반 추정에 불과함.
- 의료 챗봇 완성도를 주장할 경우, 추가 개발 단계 필요성을 무시함.

## 3. 제출 문서에서 반드시 완곡하게 표현해야 할 포인트 3가지
- 데이터 규모의 제한: "현재 323건으로 초기 분석에 적합하나, 더 큰 규모의 검증이 필요함"으로 표현하여 확장 가능성을 암시.
- 도메인 편중: "의료 대화 중심으로 특화되어 있어, 일반 대화 적용 시 추가 데이터 통합을 고려함"으로 한계를 인정.
- 품질 완성도: "기본 전처리 및 EDA를 완료하였으나, 라벨링 및 다양성 측면에서 지속적 개선이 요구됨"으로 잠재적 개선점을 언급.
