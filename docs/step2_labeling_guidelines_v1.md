# Step2 데이터셋 라벨링 가이드라인 (v1)

## 개요
본 가이드라인은 Step2 데이터셋의 감정(sentiment) 및 토픽(topic) 라벨링을 위한 표준을 정의합니다. 라벨링은 수동 검토를 원칙으로 하며, 일관성을 위해 예시와 규칙을 준수합니다. 버전 v1은 초기 데이터(350건 기반) 적용을 대상으로 합니다.

## 1. sentiment_label (감정 레이블)
감정 레이블은 텍스트의 전반적인 감정적 톤을 분류합니다. positive(긍정), neutral(중립), negative(부정)으로 구분.

### 정의
- **positive**: 만족, 긍정적 평가, 희망, 감사 등의 긍정적 표현.
- **neutral**: 사실적 진술, 질문, 정보 제공으로 감정적 편향 없음.
- **negative**: 불만, 불편, 불만족, 우려 등의 부정적 표현.

### 예시
#### positive (5개)
1. "예약이 잘 되었어요, 감사합니다." (만족 표현)
2. "친절한 응대에 기분이 좋았습니다." (긍정적 평가)
3. "빠른 진료가 가능해서 다행이에요." (희망적)
4. "약이 잘 맞아서 효과적이었어요." (긍정적 결과)
5. "다음에도 이용하고 싶습니다." (재방문 의사)

#### neutral (5개)
1. "예약은 어떻게 하나요?" (질문)
2. "진료 시간은 오전 9시부터입니다." (정보 제공)
3. "약국 위치를 알려주세요." (사실적 요청)
4. "대기 시간이 얼마나 걸리나요?" (객관적 문의)
5. "복용 방법은 하루 세 번입니다." (지침 전달)

#### negative (5개)
1. "예약이 안 돼서 불편합니다." (불만)
2. "직원이 불친절했어요." (부정적 평가)
3. "대기가 너무 길어서 짜증 나요." (불편)
4. "약이 효과가 없었어요." (불만족)
5. "가격이 너무 비싸요." (우려)

## 2. topic_label (토픽 레이블)
토픽 레이블은 텍스트의 주요 주제를 분류합니다. 의료 예약 챗봇 맥락에 맞춰 정의.

### 정의 및 예시
- **reservation**: 예약 관련 (예약 신청, 변경, 취소). 예: "예약하고 싶어요.", "예약 시간을 바꿔주세요."
- **policy**: 정책/규정 관련 (운영 시간, 규칙, 자격). 예: "휴일 진료는 가능한가요?", "예약 취소 정책은 무엇인가요?"
- **location**: 위치/접근성 관련 (주소, 교통, 찾는 법). 예: "병원 위치가 어디예요?", "주차장은 있나요?"
- **prep**: 준비/요구사항 관련 (준비물, 사전 준비). 예: "검진 전 공복인가요?", "신분증 지참해야 하나요?"
- **pickup**: 픽업/수령 관련 (약 수령, 결과 수령). 예: "약은 어디서 받나요?", "검사 결과는 어떻게 가져가나요?"
- **wait**: 대기 관련 (대기 시간, 순서). 예: "대기 시간이 얼마나 걸리나요?", "번호표는 어떻게 받나요?"
- **staff**: 직원/서비스 관련 (친절도, 응대). 예: "직원이 친절했어요.", "상담원이 도움을 주었어요."
- **price**: 가격/비용 관련 (요금, 보험). 예: "진료비는 얼마인가요?", "보험 적용되나요?"
- **process**: 절차/흐름 관련 (진료 과정, 단계). 예: "진료 순서는 어떻게 되나요?", "예약 후 다음 단계는?"

## 3. 애매한 문장 처리 규칙
애매한 문장은 여러 토픽이나 감정이 혼합된 경우 우선순위를 적용하여 단일 라벨 선택. 감정은 텍스트 전체 톤 기준, 토픽은 주요 의도 기준.

### 우선순위 규칙
1. **주요 의도 우선**: 문장의 핵심 질문/요청에 따라 토픽 결정 (예: "예약하고 싶어요, 위치도 알려주세요." → reservation).
2. **감정 강도 우선**: 부정적 표현이 있으면 negative, 긍정적이면 positive, 혼합 시 neutral (예: "친절했지만 대기가 길었어요." → neutral).
3. **문맥 고려**: 이전/다음 문장 참고 가능하나, 독립적 라벨링 원칙.
4. **다중 토픽 시 첫 번째**: 여러 토픽이 섞이면 텍스트 순서상 첫 번째 주요 토픽 선택 (예: "예약하고 약 받는 법." → reservation).
5. **불명확 시 neutral/reservation**: 감정 불명 시 neutral, 토픽 불명 시 reservation (기본값).

## 4. 라벨 품질 체크 기준
라벨링 후 품질 검증을 통해 일관성과 정확성 확보.

- **결측률 목표**: 0% (모든 행에 라벨 필수).
- **일관성 체크**: 샘플 10% 재라벨링, 일치율 90% 이상.
- **정확성 기준**: 정의 및 예시에 부합, 애매 시 규칙 적용.
- **검증 방법**: 교차 검토(2인 이상), 자동 도구로 감정 분석 비교.
- **보고**: 결측 수, 불일치율, 개선 사항 기록.
