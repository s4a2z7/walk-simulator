# Step 3 — 프롬프트 평가 기준(루브릭) v0

## 공통 원칙
- **입력에 없는 내용 생성 금지(환각 금지)**: 사실 추가/추측/과장 시 감점
- **형식 준수**: 요구한 JSON 스키마/필드/타입을 지키지 않으면 실패
- **개인정보/민감정보 생성 금지**

---

## 1) 리뷰 요약(review_summary)

### 평가 항목(0~2점)
| 항목 | 0점 | 1점 | 2점 |
|---|---|---|---|
| 정확도(핵심 보존) | 핵심 누락/왜곡 | 일부만 반영 | 핵심 정보 정확히 반영 |
| 간결성 | 장황/중복 | 다소 길음 | 1~2문장 + 2~4 bullet로 깔끔 |
| 감정 추정 근거 | 근거 불명/오류 | 애매 | 입력 근거와 일치 |
| 형식(JSON) | 깨짐/추가 텍스트 | 경미한 형식 문제 | JSON만, 스키마 완전 준수 |

### 합격 기준(초안)
- 총점 6/8 이상 AND 형식(JSON) 2점

---

## 2) 대화 요약(dialog_summary)

### 평가 항목(0~2점)
| 항목 | 0점 | 1점 | 2점 |
|---|---|---|---|
| 요약 품질 | 흐름 파악 불가 | 일부 반영 | 2~4문장으로 핵심 정리 |
| user_intent 정확성 | 의도 오판 | 애매 | 질문/요구가 정확 |
| assistant_response 정확성 | 근거 없음 | 일부 | 입력 내 답변 요지 정확 |
| next_actions | 환각/불필요 | 일부 근거 | 입력 근거 기반 1~3개 |
| 형식(JSON) | 깨짐/추가 텍스트 | 경미한 형식 문제 | JSON만, 스키마 완전 준수 |

### 합격 기준(초안)
- 총점 8/10 이상 AND 형식(JSON) 2점

## 3) 키워드 추출(keyword_extraction)

|---|---|---|---|
