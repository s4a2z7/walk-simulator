# 데모 시나리오(3종) — 의료업체 통합 예약 챗봇(1분 시연용)

## 시나리오 1) 병원 예약 추천(핸드오프)
**목표**: 병원 후보 추천 → 1개 선택 → 연락/링크 제공(핸드오프)

- 사용자: “오늘 저녁에 병원 예약 추천받고 싶어요.”
- 챗봇: “어느 유형으로 예약할까요? (병원/약국/검진센터)”
- 사용자: “병원”
- 챗봇: “지역이나 병원명을 알려주세요. (예: 강남역 / ○○내과)”
- 사용자: “강남역 근처”
- 챗봇: “후보를 선택해주세요. 1) A내과  2) B이비인후과  3) C의원”
- 사용자: “1번”
- 챗봇: “원하시는 날짜/시간대가 있나요? (예: 오늘 저녁/내일 오전)”
- 사용자: “오늘 저녁”
- 챗봇: “좋아요. 후보를 추천드릴게요. 1) A내과(강남)  2) B이비인후과(강남)  3) C의원(강남)”
- 사용자: “1번”
- 챗봇: “A내과로 예약할 수 있도록 연락 경로를 드릴게요.  
  - 전화: (번호/링크)  
  - 홈페이지: (링크)  
  - 운영시간/준비사항: (요약)”

---

## 시나리오 2) 약국 수령/문의 추천(핸드오프)
**목표**: 약국 후보 추천 → 1개 선택 → 전화/경로 제공

- 사용자: “약국에서 처방약 수령 예약하고 싶어요.”
- 챗봇: “지역이나 약국명을 알려주세요.”
- 사용자: “홍대입구 근처”
- 챗봇: “후보를 선택해주세요. 1) A약국  2) B약국  3) C약국”
- 사용자: “2번”
- 챗봇: “방문 예정 시간대가 있나요? (예: 지금/오늘 오후/내일 오전)”
- 사용자: “지금”
- 챗봇: “후보를 추천드릴게요. 1) A약국  2) B약국  3) C약국”
- 사용자: “2번”
- 챗봇: “B약국으로 연결해드릴게요.  
  - 전화: (번호/링크)  
  - 위치: (지도 링크)  
  - 참고: 처방 조제/수령 가능 여부는 약국에 확인해주세요.”

---

## 시나리오 3) 추천 조건 변경 포함(재추천)
**목표**: 기존 추천 불만족 → 조건 변경 → 재추천

- 사용자: “추천받은 곳이 너무 멀어요. 가까운 곳으로 바꿔줘요.”
- 챗봇: “좋아요. 지역을 더 좁혀볼까요? (예: 강남역 1km 이내 / 역삼역 근처)”
- 사용자: “강남역 1km 이내”
- 챗봇: “재추천 결과입니다. 1) D의원  2) E내과  3) F이비인후과”
- 사용자: “2번”
- 챗봇: “E내과로 연결해드릴게요. 전화/홈페이지/위치 링크를 드립니다.”
