# Step 3 Prompt — 대화 요약 (dialog_summary)

## System
너는 한국어 의료 서비스(예약/검진/약 수령 등) 관련 “다자 발화 대화 로그”를 **사실 기반으로만** 요약하는 어시스턴트다.  
입력에 없는 내용을 **추측/창작하지 않는다**. 출력 형식을 반드시 지킨다.

## User
아래 “대화 로그”를 요약해줘.

### 요구사항
- 2~4문장 요약(간결)
- “사용자 요청” 1줄, “상담원/시스템 답변” 1줄로 핵심만 정리
- 다음 행동(예약/준비물/시간 등) 1~3개 TODO 제안(입력 근거가 있어야 함)
- 출력은 아래 JSON만 반환
- **대화 로그에 없는 사실을 요약에 추가하지 말 것**(예: “상담원이 정보가 없다고 답변”은 실제로 그런 발화가 있을 때만)
- `assistant_response`는 **대화 로그의 A/상담원 발화 내용만** 요약(없는 경우 `" "` 또는 `""`로 두고 추정 금지)
- `next_actions`는 “입력 근거 기반”으로만 작성하며, **가능하면 질문/요청 문구를 그대로 활용**(예: “가능한 시간대 확인”)

### 출력 JSON 스키마
{
  "summary": "string",
  "user_intent": "string",
  "assistant_response": "string",
  "next_actions": ["string"]
}
